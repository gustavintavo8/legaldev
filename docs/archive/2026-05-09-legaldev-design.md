# LegalDev RAG — Design Spec
Date: 2026-05-09

## Overview

LegalDev is a RAG-based API that, given a structured questionnaire about a software project, returns the applicable Spanish and European legal regulations with concrete technical implications for developers. Targeted at individual developers in Spain. Open source, with clear disclaimers that it is informative guidance, not legal advice.

---

## Stack

| Layer | Technology |
|-------|-----------|
| Runtime | Python 3.11+ |
| API framework | FastAPI |
| Vector store | ChromaDB (local, persistent on disk) |
| Embeddings | `sentence-transformers` — `paraphrase-multilingual-MiniLM-L12-v2` |
| LLM | Groq API — `meta-llama/llama-4-scout-17b-16e-instruct` (configurable via env) |
| Deploy | Railway via Dockerfile |
| Local dev | docker-compose |

## Architecture

```
Client ──POST /analyze──▶ main.py (FastAPI)
                           lifespan: embeddings, vectorstore, groq_client → app.state
                                │
                                ▼
                          rag.run_pipeline(input, state)
                           1. build semantic query string
                           2. embed + ChromaDB retrieval (top-8 chunks)
                           3. build system prompt + user message
                           4. Groq API call
                           5. assemble RAGResponse
                                │
                    ┌───────────┴───────────┐
                    ▼                       ▼
             ChromaDB (local)         Groq API (remote)
             ./chroma_db              llama-4-scout
```

Offline indexing (run once before Docker build):
```
docs/*.pdf → PyPDFLoader → RecursiveCharacterTextSplitter → HuggingFaceEmbeddings → ChromaDB
```

## Component Design

### `config.py`

`Settings` class using `pydantic-settings` (`BaseSettings`). Values loaded from `.env` automatically:

```python
GROQ_API_KEY: str
GROQ_MODEL: str = "meta-llama/llama-4-scout-17b-16e-instruct"
CHROMA_DB_PATH: str = "./chroma_db"
DOCS_PATH: str = "./docs"
TOP_K_CHUNKS: int = 8
```

### `models.py`

**`QuestionnaireInput`** — 16 fields across 4 sections:
- Section 1 (project): `tipo_proyecto`, `descripcion_breve`, `tiene_usuarios_registrados`, `acceso_publico`
- Section 2 (data/users): `tipos_datos_personales`, `usuarios_menores`, `usuarios_ue`, `transferencia_datos_terceros`
- Section 3 (tech): `usa_ia`, `tipo_ia`, `usa_cookies`, `monetizacion`, `contenido_digital`
- Section 4 (context): `ccaa`, `es_empresa`, `colegiado`

**`RAGResponse`**:
```python
respuesta_completa: str      # full LLM text
normativas_detectadas: list[str]  # unique source filenames (without .pdf) from retrieved chunks
chunks_utilizados: int       # number of chunks retrieved (≤ TOP_K_CHUNKS)
disclaimer: str              # hardcoded Spanish disclaimer constant
```

### `ingest.py`

Standalone script (`python ingest.py`). Not imported by the app.

1. Load env vars from `.env`
2. Wipe existing ChromaDB collection if present
3. For each PDF in `DOCS_PATH`:
   - Load with `PyPDFLoader`
   - Split: `RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)`
   - Assign `source` (filename) and `doc_type` metadata
   - Log: `INFO: Indexing <filename> → N chunks`
4. Embed all chunks: `HuggingFaceEmbeddings(model_name="paraphrase-multilingual-MiniLM-L12-v2")`
5. Persist to ChromaDB via `Chroma.from_documents()`
6. Log: `INFO: Indexing complete — M total chunks across N documents`

**`doc_type` mapping:**

| File | `doc_type` |
|------|-----------|
| RGPD.pdf | `normativa_europea` |
| EU AI Act.pdf | `normativa_europea` |
| Directiva NIS2.pdf | `normativa_europea` |
| Directiva de Responsabilidad por Productos con IA.pdf | `normativa_europea` |
| LOPDGDD.pdf | `normativa_española` |
| ENS.pdf | `normativa_española` |
| LSSI.pdf | `normativa_española` |
| Ley de Propiedad Intelectual.pdf | `normativa_española` |
| Código Ético y Deontológico CCII.pdf | `deontologia` |
| Any file containing "AEPD" in the name | `guia_aepd` |

### `rag.py`

Exposes one public function: `run_pipeline(input: QuestionnaireInput, state: fastapi.datastructures.State) -> RAGResponse`

**Step 1 — Semantic query construction:**

Build a descriptive Spanish string from questionnaire fields. Maps:
- `tipo_proyecto` → literal value
- `tipos_datos_personales` → joined list
- `usa_ia=True` + `tipo_ia` → "inteligencia artificial {tipo_ia}"
- `usa_cookies=True` → "cookies"
- `monetizacion` → included if not "ninguna"
- `ccaa` → appended
- `usuarios_ue=True` → "usuarios Unión Europea"
- `usuarios_menores=True` → "usuarios menores de edad"
- `transferencia_datos_terceros=True` → "transferencia datos terceros"
- `descripcion_breve` → appended

**Step 2 — Retrieval:**

```python
docs = state.vectorstore.similarity_search(query, k=settings.TOP_K_CHUNKS)
```

**Step 3 — Prompt construction:**

System prompt (hardcoded constant):
```
Eres LegalDev, un asistente especializado en normativa legal aplicable a proyectos
de software en España y la Unión Europea.

Tu misión es analizar el contexto de un proyecto de software y explicar qué normativas
legales le aplican, con implicaciones técnicas concretas y accionables para el developer.

Reglas:
- Responde siempre en español
- Sé específico y técnico: no digas "debes cumplir el RGPD", di qué tienes que implementar exactamente
- Organiza la respuesta por normativa aplicable
- Incluye siempre el disclaimer al final
- No inventes normativas que no estén en el contexto proporcionado

Disclaimer obligatorio al final de cada respuesta:
"⚠️ Esta información es orientativa y no constituye asesoramiento legal. Para decisiones
con impacto legal, consulta con un abogado especializado en derecho digital."
```

User message: summary of questionnaire answers + numbered retrieved chunks.

**Step 4 — LLM call:** `state.groq_client.invoke([system_msg, human_msg])`

**Step 5 — Response assembly:**
- `respuesta_completa` = LLM output text
- `normativas_detectadas` = `list(set(doc.metadata["source"].replace(".pdf","") for doc in docs))`
- `chunks_utilizados` = `len(docs)`
- `disclaimer` = hardcoded disclaimer constant

**Error handling:** any exception from Groq → `raise HTTPException(status_code=503, detail="LLM service unavailable. Please try again later.")`

### `main.py`

```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    app.state.embeddings = HuggingFaceEmbeddings(model_name="paraphrase-multilingual-MiniLM-L12-v2")
    app.state.vectorstore = Chroma(persist_directory=settings.chroma_db_path, embedding_function=app.state.embeddings)
    app.state.groq_client = ChatGroq(api_key=settings.groq_api_key, model_name=settings.groq_model)
    yield
```

**Endpoints:**

| Method | Path | Response |
|--------|------|----------|
| `GET` | `/` | `{"name": "LegalDev", "version": "0.1.0", "description": "Asistente RAG de normativa legal para developers en España"}` |
| `GET` | `/health` | `{"status": "ok", "docs_indexed": N}` via `vectorstore._collection.count()` |
| `POST` | `/analyze` | `QuestionnaireInput` → `RAGResponse` |

**CORS:** `CORSMiddleware(allow_origins=["*"], allow_methods=["*"], allow_headers=["*"])`

**Logging:** `logging.basicConfig(level=logging.INFO)` at startup.

**Authentication:** none.

## Data Flow (end-to-end)

```
POST /analyze
  │
  ├─ FastAPI validates QuestionnaireInput (Pydantic)
  │
  ├─ rag.run_pipeline(input, request.app.state)
  │     ├─ build query string from input fields
  │     ├─ vectorstore.similarity_search(query, k=8)  → list[Document]
  │     ├─ build [SystemMessage, HumanMessage]
  │     ├─ groq_client.invoke(messages)               → AIMessage
  │     └─ assemble RAGResponse
  │
  └─ return RAGResponse (JSON)
```

## Deployment

**Pre-build steps (local):**
1. Add PDFs to `docs/`
2. `python app/ingest.py` — generates `chroma_db/`
3. `docker build .` — bakes `chroma_db/` into image
4. Push to Railway

**Dockerfile:**
```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY app/ ./app/
COPY chroma_db/ ./chroma_db/
EXPOSE 8000
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**docker-compose.yml** mounts `chroma_db/` and `docs/` as volumes so `ingest.py` can be re-run outside the container during local dev.

**`.env.example`:**
```
GROQ_API_KEY=your_groq_api_key_here
GROQ_MODEL=meta-llama/llama-4-scout-17b-16e-instruct
CHROMA_DB_PATH=./chroma_db
DOCS_PATH=./docs
TOP_K_CHUNKS=8
```

## Key Constraints

- Works fully offline once indexed (ChromaDB local + sentence-transformers downloaded on first run)
- Embeddings generated once by `ingest.py`; app only does retrieval
- App does not re-index on startup even if `chroma_db/` is missing (it will fail with a clear error)
- `ingest.py` always wipes and rebuilds when run manually
- All logging via Python `logging` module, no `print()`
- No authentication on any endpoint
